# Настройка админ-панели CardVault

## 🔧 Настройка переменных окружения

Создайте файл `.env.local` в корне проекта и добавьте следующие переменные:

```env
# Cloudflare R2 Configuration
NEXT_PUBLIC_R2_ENDPOINT=https://your-account-id.r2.cloudflarestorage.com
NEXT_PUBLIC_R2_ACCESS_KEY_ID=your-access-key-id
NEXT_PUBLIC_R2_SECRET_ACCESS_KEY=your-secret-access-key
NEXT_PUBLIC_R2_PUBLIC_URL=https://pub-your-domain.r2.dev
```

## 🚀 Настройка Cloudflare R2

1. **Создайте R2 бакет:**
   - Зайдите в панель Cloudflare
   - Перейдите в раздел R2 Object Storage
   - Создайте новый бакет с именем `cards`

2. **Получите API токены:**
   - В разделе R2 нажмите "Manage R2 API tokens"
   - Создайте новый токен с правами на чтение и запись
   - Скопируйте Access Key ID и Secret Access Key

3. **Настройте публичный домен (опционально):**
   - В настройках бакета включите "Public Access"
   - Настройте custom domain или используйте предоставленный домен

## 🔥 Настройка Firebase

Firebase уже настроен с вашими данными:
- Project ID: `cards-f8d41`
- Authentication включен
- Firestore настроен

### Создание администратора

1. Зайдите в Firebase Console
2. Перейдите в Authentication
3. Добавьте нового пользователя с email и паролем
4. Этот пользователь сможет войти в админ-панель

## 📱 Доступ к админ-панели

После настройки переменных окружения:

1. Перейдите на `/admin/login`
2. Введите email и пароль администратора
3. После входа вы попадете в главную панель админа

## 🔒 Безопасность

- Доступ к админ-панели защищен Firebase Authentication
- Все загрузки файлов проходят валидацию
- R2 настроен на загрузку только изображений
- Автоматическое перенаправление неавторизованных пользователей

## 🚦 Статусы карточек

- **active** - отображается на сайте
- **inactive** - скрыта от пользователей
- **draft** - черновик (не отображается)

## 📋 Разделы карточек

- **new-releases** - Предложения недели
- **best-sellers** - Лидеры продаж
- **new-in-catalog** - Новое в каталоге
- **catalog** - Общий каталог

## 🔄 Синхронизация данных

Все данные сохраняются в Firestore и автоматически синхронизируются между админ-панелью и сайтом в реальном времени.
